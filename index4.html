Вопросы

Основы тестирования

Что такое Тестирование (Testing)?
Тестирование (Testing) — это процесс проверки продукта на соответствие требованиям, выявление дефектов и обеспечение качества.
Какие виды Тестирования (Testing) существуют?
Основные виды: Функциональное тестирование (Functional Testing), Нефункциональное тестирование (Non-functional Testing), Ручное тестирование (Manual Testing), Автоматизированное тестирование (Automation Testing).
Что такое Black-box тестирование?
Black-box тестирование — метод тестирования, при котором тестировщик не имеет доступа к исходному коду и работает только через интерфейс.
Что такое White-box тестирование?
White-box тестирование — метод тестирования, при котором тестировщик имеет доступ к исходному коду и может проверять внутреннюю логику программы.
Что такое Smoke-тестирование?
Smoke-тестирование — быстрая проверка основных функций системы после установки или обновления.
Что такое Regression-тестирование?
Regression-тестирование — повторное тестирование после внесения изменений для проверки отсутствия новых ошибок.

Техники тест-дизайна

Что такое Тестирование граничных значений (Boundary Value Analysis)?
Тестирование граничных значений (Boundary Value Analysis) — метод тестирования, при котором проверяются значения на границах допустимых диапазонов.
Что такое Эквивалентное разделение (Equivalence Partitioning)?
Эквивалентное разделение (Equivalence Partitioning) — метод тестирования, при котором входные данные делятся на классы эквивалентности.
Что такое Error Guessing?
Error Guessing — метод тестирования, при котором тестировщик предполагает возможные ошибки и создает тест-кейсы для их проверки.
Что такое Decision Table Testing?
Decision Table Testing — метод тестирования, основанный на анализе таблиц решений, где определяются правила и условия.
Что такое State Transition Testing?
State Transition Testing — метод тестирования, при котором проверяются переходы между состояниями системы.

Методы тестирования

Что такое Функциональное тестирование (Functional Testing)?
Функциональное тестирование (Functional Testing) — проверка того, что система выполняет свои функции в соответствии с требованиями.
Что такое Нефункциональное тестирование (Non-functional Testing)?
Нефункциональное тестирование (Non-functional Testing) — проверка характеристик системы, таких как производительность, безопасность, совместимость.
Что такое Ручное тестирование (Manual Testing)?
Ручное тестирование (Manual Testing) — тестирование, проводимое вручную без использования автоматизации.
Что такое Автоматизированное тестирование (Automation Testing)?
Автоматизированное тестирование (Automation Testing) — тестирование, проводимое с помощью специальных инструментов и скриптов.
Что такое Performance Testing?
Performance Testing — проверка скорости, стабильности и эффективности работы системы.

Баг-репорты и документация

Что такое Баг-репорт (Bug Report)?
Баг-репорт (Bug Report) — документ, описывающий найденный дефект, включая шаги воспроизведения, фактический и ожидаемый результаты.
Какие элементы должны быть в Баг-репорте (Bug Report)?
ID бага, описание, предусловия, шаги воспроизведения, фактический результат, ожидаемый результат, окружение, уровень критичности.
Что такое Тест-план (Test Plan)?
Тест-план (Test Plan) — документ, содержащий описание цели, области покрытия, рисков, ресурсов и сроков тестирования.
Что такое Test Case?
Test Case — набор шагов, входных данных и ожидаемых результатов для проверки конкретной функции.
Что такое Матрица трассировки (Traceability Matrix)?
Матрица трассировки (Traceability Matrix) — матрица, связывающая требования с тест-кейсами для оценки покрытия.

Cypress — современный инструмент для автоматизации тестирования веб-приложений.

Тестирование безопасности

Что такое SQL-инъекция (SQL Injection)?
SQL-инъекция (SQL Injection) — атака, при которой злоумышленник вводит вредоносный SQL-код через пользовательский ввод.
Что такое XSS-атака (Cross-Site Scripting)?
XSS-атака (Cross-Site Scripting) — атака, при которой злоумышленник внедряет вредоносный JavaScript-код в веб-страницу.
Что такое CSRF-атака (Cross-Site Request Forgery)?
CSRF-атака (Cross-Site Request Forgery) — атака, при которой злоумышленник заставляет пользователя выполнить нежелательное действие.
Что такое OWASP Top 10?
OWASP Top 10 — список наиболее распространенных уязвимостей веб-приложений.
Что такое Authorization (Авторизация)?
Authorization (Авторизация) — процесс определения прав доступа пользователя.

Баг-репорты и документация

Вопросы
Что такое Баг-репорт (Bug Report)?
Баг-репорт (Bug Report) — документ, описывающий найденный дефект, включая шаги воспроизведения, фактический и ожидаемый результаты.
Какие элементы должны быть в Баг-репорте (Bug Report)?
ID бага, описание, предусловия, шаги воспроизведения, фактический результат, ожидаемый результат, окружение, уровень критичности.
Что такое Тест-план (Test Plan)?
Тест-план (Test Plan) — документ, содержащий описание цели, области покрытия, рисков, ресурсов и сроков тестирования.
Пример структуры Тест-плана (Test Plan)?
Введение, область покрытия, входные/выходные критерии, стратегия тестирования, распределение задач, риски.
Что такое Тест-кейс (Test Case)?
Тест-кейс (Test Case) — набор шагов, входных данных и ожидаемых результатов для проверки конкретной функции.
Пример структуры Тест-кейса (Test Case)?
ID, название, предусловия, шаги, входные данные, ожидаемый результат.
Что такое Чек-лист (Checklist)?
Чек-лист (Checklist) — список проверок, который помогает быстро протестировать основные функции системы.
Чем Чек-лист (Checklist) отличается от Тест-кейса (Test Case)?
Чек-лист менее детализирован, чем тест-кейс, и используется для быстрого тестирования.

Что такое Scrum?
Scrum — методология управления проектами, основанная на коротких спринтах (обычно 2 недели).
Пример Scrum-процесса?
Создание бэклога → Планирование спринта → Ежедневные встречи → Демонстрация результатов.
Что такое Kanban?
Kanban — методология управления проектами, основанная на потоке задач.
Пример Kanban-процесса?
Использование доски задач для отслеживания статуса каждой задачи.
Что такое Юзер-стори (User Story)?
Юзер-стори (User Story) — описание функциональности с точки зрения пользователя.
Пример Юзер-стори (User Story)?
Как пользователь, я хочу иметь возможность сбросить пароль, чтобы восстановить доступ к аккаунту.
Что такое Спринт (Sprint)?
Спринт (Sprint) — короткий временной интервал (обычно 1–4 недели), в течение которого выполняется работа над задачами.
Пример Спринта (Sprint)?
Разработка и тестирование функции восстановления пароля за 2 недели.
Что такое Бэклог (Backlog)?
Бэклог (Backlog) — список задач, которые нужно выполнить в проекте.
Пример Бэклога (Backlog)?
Список функций, включающий регистрацию, авторизацию и восстановление пароля.
Что такое Ежедневная встреча (Daily Standup)?
Ежедневная встреча (Daily Standup) — короткая встреча команды для обсуждения прогресса и проблем.
Пример Ежедневной встречи (Daily Standup)?
Обсуждение выполненных задач, планов на день и возможных препятствий.
Что такое Ретроспектива (Retrospective)?
Ретроспектива (Retrospective) — встреча после завершения спринта для анализа процесса.
Пример Ретроспективы (Retrospective)?
Обсуждение того, что удалось сделать хорошо, что можно улучшить, и какие проблемы возникли.
Что такое Планирование спринта (Sprint Planning)?
Планирование спринта (Sprint Planning) — встреча для определения задач текущего спринта.
Пример Планирования спринта (Sprint Planning)?
Выбор задач из бэклога для реализации в следующем спринте.
Что такое Демонстрация результатов (Sprint Review)?
Демонстрация результатов (Sprint Review) — встреча для демонстрации результатов спринта заказчику.
Пример Демонстрации результатов (Sprint Review)?
Показ новой функции восстановления пароля заказчику.
Что такое Пользовательские истории (User Stories)?
Пользовательские истории (User Stories) — описание требований с точки зрения пользователя.
Пример Пользовательских историй (User Stories)?
Как пользователь, я хочу иметь возможность сбросить пароль, чтобы восстановить доступ к аккаунту.
Что такое Эпик (Epic)?
Эпик (Epic) — большая пользовательская история, которая делится на несколько smaller задач.
Пример Эпика (Epic)?
Эпик "Улучшение безопасности": включает задачи по добавлению двухфакторной аутентификации и шифрованию данных.
Что такое С STORY POINTS (Story Points)?
Story Points — относительная оценка сложности задачи в Agile-проектах.
Пример использования Story Points?
Оценка задачи "Добавить функцию восстановления пароля" в 5 story points.
Что такое Бёрндаун диаграмма (Burndown Chart)?
Бёрндаун диаграмма (Burndown Chart) — график, показывающий оставшийся объем работы в спринте.
Пример Бёрндаун диаграммы (Burndown Chart)?
График, демонстрирующий уменьшение объема задач за время спринта.
Что такое Валидация продукта (Product Validation)?
Валидация продукта (Product Validation) — проверка того, что продукт удовлетворяет потребностям пользователя.
Пример Валидации продукта (Product Validation)?
Проверка того, что новая функция восстановления пароля удобна для пользователей.
Что такое Верификация продукта (Product Verification)?
Верификация продукта (Product Verification) — проверка соответствия продукта требованиям.
Пример Верификации продукта (Product Verification)?
Проверка того, что функция восстановления пароля работает согласно техническому заданию.
Что такое Инкремент (Increment)?
Инкремент (Increment) — готовый и проверенный результат работы спринта.
Пример Инкремента (Increment)?
Функция восстановления пароля, готовая к выпуску после завершения спринта.
Что такое Product Owner?
Product Owner — человек, ответственный за приоритизацию задач и представление интересов заказчика.
Пример роли Product Owner?
Product Owner определяет, какие функции должны быть реализованы в следующем спринте.
Что такое Scrum Master?
Scrum Master — человек, помогающий команде следовать процессам Scrum.
Пример роли Scrum Master?
Scrum Master организует ежедневные встречи и устраняет препятствия для команды.
Что такое Sprint Goal?
Sprint Goal — цель спринта, которую команда должна достичь.
Пример Sprint Goal?
Добавить функцию восстановления пароля и улучшить безопасность входа.
Что такое Definition of Done (DoD)?
Definition of Done (DoD) — список критериев, которым должна соответствовать задача для считаться завершенной.
Пример Definition of Done (DoD)?
Задача считается завершенной, если она протестирована, документирована и готова к выпуску.
Что такое Velocity?
Velocity — метрика, показывающая скорость выполнения задач командой за спринт.
Пример Velocity?
Команда выполнила 30 story points за последний спринт.
